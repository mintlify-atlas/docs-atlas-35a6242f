This guide walks you through a complete end-to-end setup: installing Clicky, initializing it, and tracking your first events.

<Steps>
  <Step title="Install Clicky">
    Add Clicky to your project using a script tag or your package manager.

    <CodeGroup>

    ```html script tag
    <script src="https://cdn.clicky.io/clicky.min.js"></script>
    ```

    ```bash npm
    npm install @clicky/web
    ```

    ```bash yarn
    yarn add @clicky/web
    ```

    ```bash pnpm
    pnpm add @clicky/web
    ```

    </CodeGroup>
  </Step>
  <Step title="Initialize Clicky">
    Call `clicky.init()` as early as possible in your app — ideally before any other JavaScript runs. Pass your site ID from the Clicky dashboard.

    ```javascript
    clicky.init({
      siteId: "YOUR_SITE_ID",
    });
    ```

    Once initialized, Clicky is active. You don't need to do anything else to start collecting data.

    <Note>
      Replace `"YOUR_SITE_ID"` with the ID from your Clicky dashboard. Each site you add gets a unique ID.
    </Note>
  </Step>
  <Step title="Auto-tracking is active">
    After `clicky.init()` runs, Clicky automatically tracks the following without any extra code:

    - **Clicks** — every click on any element across the page, enriched with element metadata
    - **Page views** — the current page URL and title are recorded on initialization

    You can verify this is working by opening the **Live Events** panel in your Clicky dashboard. Clicks you make on the page will appear within seconds.

    <Tip>
      Add `data-clicky-label="my-button"` to any element to give it a human-readable name in your dashboard. Clicky picks this up automatically.
    </Tip>
  </Step>
  <Step title="Track a custom event">
    Use `clicky.track()` to record any user action that isn't a raw click — form submissions, video plays, feature usage, and so on.

    ```javascript
    // Track a simple event
    clicky.track("signed_up");

    // Track an event with properties
    clicky.track("plan_selected", {
      plan: "pro",
      billing: "annual",
      price: 99,
    });
    ```

    The first argument is the event name (a string). The second argument is an optional object of properties — any key-value pairs you want to associate with the event.
  </Step>
  <Step title="Identify a user">
    By default, Clicky tracks users anonymously. Once a user signs in or completes an action that reveals their identity, call `clicky.identify()` to link all subsequent events to that user.

    ```javascript
    clicky.identify("user_123", {
      name: "Jane Doe",
      email: "jane@example.com",
      plan: "pro",
    });
    ```

    You can call `clicky.identify()` at any point — Clicky will retroactively associate prior events from the same session with the identified user.
  </Step>
</Steps>

## Complete example

Here's everything together in a single setup file:

```javascript app.js
import clicky from "@clicky/web";

// 1. Initialize
clicky.init({
  siteId: "abc123xyz",
});

// 2. Identify the user once they log in
function onLogin(user) {
  clicky.identify(user.id, {
    name: user.name,
    email: user.email,
    plan: user.plan,
  });
}

// 3. Track a custom event
document.querySelector("#upgrade-btn").addEventListener("click", () => {
  clicky.track("upgrade_clicked", {
    source: "navbar",
    plan: "pro",
  });
});

// 4. Track a page view when navigating in an SPA
function onRouteChange(pageName) {
  clicky.page(pageName, { path: window.location.pathname });
}
```
