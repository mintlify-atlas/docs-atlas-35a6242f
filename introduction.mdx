Clicky lets you capture clicks, page views, and custom user interactions with minimal setup. It instruments your DOM automatically, so you start collecting meaningful data the moment you initialize it — no manual tagging required.

## How it works

When you call `clicky.init()`, Clicky attaches a global event listener to the document. It captures click events as they bubble up the DOM, enriching each event with contextual metadata: the element type, text content, `data-clicky-*` attributes, and the current page URL.

Events are batched and sent to the Clicky ingestion endpoint at regular intervals, reducing network overhead. Sessions are managed client-side using a combination of cookies and `sessionStorage`, with automatic expiry after 30 minutes of inactivity.

```
User click → DOM event listener → Event enrichment → Batch queue → Clicky ingestion API
```

You can extend this pipeline at any point: add custom properties to every event via `clicky.init()`, identify users mid-session with `clicky.identify()`, or manually track events that don't involve DOM interactions using `clicky.track()`.
